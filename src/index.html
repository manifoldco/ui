Connection

The connection is necessary to wrap all child web components, and must live at the top level. This sets the base URL (production or staging) for all children‚Äôs API calls.

Usage

<!-- Production -->
<manifold-connection>
  <manifold-marketplace></manifold-marketplace>
</manifold-connection>

<!-- Staging -->
<manifold-connection env="stage">
  <manifold-product product-label="ant-hill-stage"></manifold-product>
</manifold-connection>

The <manifold-connection> component is only needed once, at the top level (but can be re-used, if for some reason multiple envs are needed). Children components can be nested far down into the tree and they‚Äôll still work (even surrounded by React components!).

If you omit the env property, manifold-connection will point to production by default.

Navigation

When users click on a product card, you expect something to happen, right? By default, service cards will emit a manifold-serviceCard-click custom event whenever a user clicks anywhere on a card. You can listen for it like so, and use this value to navigate client-side or perform some other action of your choice: 

document.addEventListener('manifold-serviceCard-click', { detail: { label } } => {
  alert(`You clicked the card for ${label}`);
});

Alternately, if you‚Äôd like the service cards to be plain, ol‚Äô <a> tags, you can specify a link-format attribute, where :product will be substituted with each product‚Äôs URL-friendly slug: 

<manifold-marketplace link-format="/product/:product" />
<!-- <a href="/product/jawsdb-mysql"> -->

Handling Events in React

Attaching listeners to custom components in React requires the use of refs. Example: 

marketplaceLoaded(node) {
  node.addEventListener("manifold-serviceCard-click", ({ detail: { label } }) => {
    alert(`You clicked the card for ${label}`);
  });
}

render() {
  return <manifold-marketplace ref={this.marketplaceLoaded} />;
}

Events are dispatched on the manifold-planUpdated custom event. To listen for that, listen for the event on document like so:

  console.log(detail);
});
// { id: "2357v8j36f5h866c32ddwwjxvfe8j", label: "nvidia-1080ti-100gb-ssd", product: "zerosix", features: { ‚Ä¶ } } }

Navigation

The large CTA in the bottom-right is configurable. By default, this component emits a manifold-planCTA-click custom event whenever the main CTA is clicked. Listen for it like so: 

document.addEventListener(
  'manifold-productCTA-click',
  ({ detail: { product, plan, features } }) => {
    alert(
      `You clicked the CTA for the ${plan} plan on ${product} with these features: ${JSON.stringify(
        features
      )}`
    );
  }
);

To turn the CTA into an <a> tag, specify a link-format attribute, using :product, :plan, and :features as placeholders: 

<manifold-product
  product-label="aiven-redis"
  link-format="/create/:product/?plan=:plan&:features"
/>
<!-- <a href="/product/aiven-redis?plan=startup-4&cpus=1"> -->

Hiding provision button

If you would like to hide the CTA altogether, specify hide-provision-button: 

<manifold-product product-label="till" hide-provision-button />

Data Components

Unlike the UI components, data components come with no additional styling or functionality. They‚Äôre simple display items that make working with data easier. And they even have built-in loading states!

Styling

Data components do not make use of Shadow DOM (only custom element), so you‚Äôre not restricted in styling them. You should have full control over them as you would any other element. In general, we suggest styling them from the parent.

Data Product Name

Retrieve a product‚Äôs full name from a :productLabel as an unstyled text node.

Can‚Äôt remember a product‚Äôs capitalization? Want to receive automatic updates if a service rebrands? We‚Äôve got you covered with a simple name web component.

Usage

<manifold-data-product-name product-label="piio" />
<!-- Piio -->

Loading State

This data component supports an optional loading state by specifying a child slot:

  Loading‚Ä¶
</manifold-data-product-name>

The loading slot supports text as well as HTML elements.

Data Product Logo

Retrieve an unstyled <img> tag with the product‚Äôs logo from a :productLabel. Similar to <manifold-data-product-name>.

By default, the <img> tag‚Äôs alt attribute is the product name. But you can override this by specifying one of your own:
